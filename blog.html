JANUARY 31, 2019

Good morning. I've been thinking about setting this blog up for several...years?...now, but recently a few different things have aligned to make it finally happen.

The first is: I really didn't want to use any pre-existing blogging platform, I want my blog to be special. Silly? Yes, kind of. But the whole point of carolineglazer.com was to practice various web design/creation tools. The homepage was a lesson in a:hover and the display property. The inital reason I purchased the domain name was to disply the d3 visualization used for the "Pretty Peggy" Melodic Contour Charts. To slap on a generic Blogger or Squarespace blog felt like it would spoil the purity (I'm so sorry) of only HTML + CSS.

Creating a blog from scratch is not that hard, it turns out, especially if you don't think long term about what happens when you have, e.g. more than two posts. It doesn't matter, we're still on post #1! I followed this guide to set up this "blog," and I learned some pretty cool things about CSS in the process.

The shape of this blog post within this webpage is achieved via the CSS property grid-column-display which feels amazingly powerful compared to any other HTML layout tools I've used in the past. Shocking that my few half-hearted attempts to throw pages onto this site haven't resulted in a thorough working knowledge of CSS. grid-column-display is super flexible and does things like let you name the [start] and [end] points of an article anything you want and define them within the columns of the page layout. It also lets you define object widths as "spanning from the start of column 2 to the end of column 7" which feels so intuitive and lets me quickly and easily produce a cascade of images within the confines of the article tag.
